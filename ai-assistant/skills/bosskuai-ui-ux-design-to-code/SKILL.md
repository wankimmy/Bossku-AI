# BosskuAI UI/UX and Design-to-Code

Use this skill for screens, flows, and implementation-ready UI guidance.

## Workflow

1. Identify the user's goal on the screen.
2. Break the interface into components, hierarchy, and interaction states.
3. Check mobile, tablet, desktop, and edge-state behavior.
4. Consider accessibility: keyboard navigation, focus order, screen reader compatibility, color contrast, and WCAG-relevant requirements where applicable.
5. Translate the design into implementation primitives:
   - layout structure
   - components
   - states
   - data needs
   - interaction rules
6. Call out ambiguity instead of inventing silent details.

## Output expectation

- UX assessment
- screen/component breakdown
- states and interactions
- responsive notes with mobile-first considerations
- accessibility notes (keyboard, focus, a11y) where relevant
- implementation handoff guidance
