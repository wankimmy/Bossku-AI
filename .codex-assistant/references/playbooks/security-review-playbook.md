# Security Review Playbook

Use this when reviewing a feature, workflow, or system for security and operational risk.

## Workflow

1. Identify assets, actors, and trust boundaries.
2. Trace who can read, write, approve, and override.
3. Check abuse paths, privilege escalation, data leaks, and fraud angles.
4. Review auditability, observability, and recovery paths.
5. Separate confirmed risks from inferred risks.

## Output expectation

- key risks
- severity and impact
- likely abuse paths
- missing controls
- recommended mitigations
