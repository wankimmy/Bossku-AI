# Polyglot Engineering Playbook

Use this when advising across different programming languages, frameworks, or runtimes.

## Workflow

1. Identify the actual stack and maturity of the codebase.
2. Prefer framework-native patterns before generic advice.
3. Explain where guidance is universal versus stack-specific.
4. Surface migration, tooling, and operational tradeoffs.
5. Recommend the strongest approach for the actual ecosystem in use.

## Output expectation

- stack-aware guidance
- preferred local patterns
- risks and tradeoffs
- recommended implementation direction
